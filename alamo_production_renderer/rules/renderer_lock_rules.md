# Renderer Lock Rules

These rules protect the report from Claude/LLM layout drift.

## Never change during normal report generation

- Template image dimensions
- Template margins
- Header placement
- Panel positions
- Panel sizes
- Chart box coordinates
- Map box coordinates
- Font sizes
- Brand colors
- Footer position
- Static labels and headings

## Only these may change per report

- Client name
- Date range
- KPI values
- Top 30 channel names and values
- State name in map header
- Generated ZIP/ZCTA map image content
- Top ZIPs text
- Device chart content
- Gender/age chart content
- Time-of-day chart content
- Footer generated date

## Rendering rule

The final report must be generated by `render_report.py`. Claude or any other LLM may prepare data, but it must not manually place report elements.

## Validation rule

Every output must include a `.validation.txt` file. Static-region validation should show zero changed samples outside approved dynamic boxes.

## Production font rule

Dynamic report text must use the approved report font only. The current approved working font is Lato because it visually matches the supplied Base True report more closely than DejaVu Sans.

Do not silently fall back to DejaVu Sans, Pillow default, browser fonts, or any other system font for client output. If an approved font is missing, the renderer must fail before generating the report. This prevents subtle text-width, weight, and placement drift.

The validation report must list the exact font file resolved for each style.
